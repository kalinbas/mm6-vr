# MM6 Unity Viewer

This folder is now a Unity 6 project scaffold plus editor importers and a generated viewer app for the JSON packages generated by:

`python3 tools/export_mm6_outdoor_to_unity.py --map /path/to/outdoor.odm --output exported/unity/outdoor`

and:

`python3 tools/export_mm6_indoor_to_unity.py --map /path/to/dungeon.blv --output exported/unity-indoor/dungeon`

The project metadata is pinned to Unity 6 `6000.4.0f1`.

The intended end state is:

- import one or more MM6 outdoor/indoor maps into `Assets/MM6Imported`
- generate the viewer bootstrap scene and map catalog
- launch the generated Quest viewer directly into the configured map, then explore it either in desktop mode or on Quest 2 in VR

## Source Game Data

Use files from a valid local Might and Magic VI installation that you own. The importer does not download game assets; it reads your local install and generates temporary Unity-ready packages from it. Do not redistribute the extracted or generated assets.

The recommended layout is:

```text
mm6hd/
  originaldata/
    Might and Magic 6/
      MM6.exe
      data/
        Games.lod
        BITMAPS.LOD
        SPRITES.LOD
        icons.lod
        new.lod
      Sounds/
        2.mp3
        3.mp3
        ...
```

Required files:

- `data/Games.lod`: outdoor maps, indoor maps, and saved map data containers.
- `data/BITMAPS.LOD`: terrain, building, item, UI, and palette-backed bitmap data.
- `data/SPRITES.LOD`: sprite frames for monsters, decorations, trees, villagers, and effects.
- `data/icons.lod`: MM6 tables such as sprites, decorations, monsters, terrain tiles, animated texture records, and map stats.
- `data/new.lod`: optional but recommended; the scripts use it when available for patched or runtime map state data.
- `Sounds/*.mp3`: optional for geometry import, but used by the viewer soundtrack setup.

If your install is somewhere else, either copy the full game folder to `originaldata/Might and Magic 6`, or pass it explicitly:

```bash
python3 tools/build_mm6_new_sorpigal_package.py --install-dir "/path/to/Might and Magic 6"
```

To validate and generate the shared extraction cache manually, run this from the repository root:

```bash
python3 tools/prepare_mm6_install_cache.py --install-dir "originaldata/Might and Magic 6"
```

That creates:

- `generated/mm6-install/bitmaps`: PNGs extracted from `BITMAPS.LOD`.
- `generated/mm6-install/sprites`: PNGs extracted from `SPRITES.LOD` using palettes from `BITMAPS.LOD`.
- `generated/mm6-install/tables`: decoded MM6 tables from `icons.lod`.

## Outdoor What It Builds

- Terrain mesh from the MM6 height map
- Outdoor BSP model geometry with per-face bitmap materials
- Billboard decorations from sprite PNGs
- Billboarded monster previews from MM6 outdoor spawn points
- SFT-driven sprite animation for idle/standing monster loops and animated decorations
- Idle-only monster previews with standing animation where available
- Outdoor time-of-day environment: sky dome, sun, moon, ambient light, fog, and winter snow that update with the in-scene hour
- Mesh colliders for terrain and building geometry
- A simple first-person player camera spawned from the exported outdoor start heuristic
- A saved Unity scene under `Assets/MM6Imported/<map>/Scenes`

## Indoor What It Builds

- Static indoor BLV geometry grouped by sector and textured per face
- Billboard dungeon decorations from sprite PNGs
- Indoor point lights from the BLV light list
- A simple first-person player camera spawned from the exported player-start heuristic
- A saved Unity scene under `Assets/MM6Imported/<map>/Scenes`

## Viewer Workflow

1. Open this folder itself as a Unity project:
   `unity/MM6OutdoorImporter`
2. Let Unity resolve packages and compile scripts.
3. For Quest 2 builds, run:
   `Tools/MM6/Configure Quest 2 Project Settings`
   For a focused New Sorpigal-only Quest test loop, first build the single package with:
   `python3 tools/build_mm6_new_sorpigal_package.py`
   then run:
   `Tools/MM6/Prepare New Sorpigal Quest Test`
4. Import your maps:
   `Tools/MM6/Import Bundled Packages In Project`
   and/or
   `Tools/MM6/Import Bundled Indoor Packages In Project`
   For a stitched desktop test world from the original MM6 outdoor regions, first build the packages with:
   `python3 tools/build_mm6_outdoor_world_packages.py`
   then run:
   `Tools/MM6/Import Stitched Full Outdoor World`
5. Run:
   `Tools/MM6/Setup Viewer Application`
6. Open the generated bootstrap scene:
   `Assets/MM6Viewer/Scenes/MM6Viewer.unity`
7. Press Play.

The generated Quest viewer scene now opens the first catalog map directly. For the focused New Sorpigal Quest test, the Build Settings list contains `MM6Viewer.unity` followed by `oute3.unity`, so the app starts the viewer bootstrap and immediately loads New Sorpigal additively at the imported spawn point. The world-space map selector is not created in this mode.

When the viewer scene is regenerated, it also imports the first soundtrack file from the original install under:

`originaldata/Might and Magic 6/Sounds`

The current rule is numeric-first ordering, so with the vanilla MM6 install it picks `2.mp3` and loops it as background music in the viewer.

## Desktop Controls

- In map: `WASD` or `Up/Down`, mouse look
- In map: `Left/Right` rotate
- In map: `Space` jump
- In map: `Shift` sprint
- Unlock mouse cursor: `Esc`

## VR Controls

- In map: hold right trigger to aim teleport, release to teleport
- In map: left stick directional movement
- In map: left secondary button jumps
- In map: left primary button toggles directional movement on/off
- In map: right stick snap turn

The VR rig is custom and uses headset/controller tracking plus scene colliders for teleport. It keeps the MM6 world scale by converting headset meters into MM6-style world units.

## Quest 2 Setup

The project now includes:

- `com.unity.xr.management`
- `com.unity.xr.openxr`

Before building for Quest 2, run the Quest setup menu above, then verify OpenXR for Android in:

`Project Settings > XR Plug-in Management`

Then enable the Meta Quest support feature group in:

`Project Settings > XR Plug-in Management > OpenXR`

After that, switch the build target to Android and build the viewer scene plus imported map scenes from the generated Build Settings list.

## Build Quest APK

Prerequisites:

- Unity Hub has Unity `6000.4.0f1` installed.
- The Unity Android Build Support module is installed, including SDK, NDK, and OpenJDK.
- The original MM6 install is available as described in `Source Game Data`.
- Run commands from the repository root, not from the Unity project subfolder.

Fast New Sorpigal Quest build:

1. Generate the New Sorpigal package from the original install:
   `python3 tools/build_mm6_new_sorpigal_package.py --install-dir "originaldata/Might and Magic 6"`
2. Open `unity/MM6OutdoorImporter` in Unity `6000.4.0f1`.
3. Run:
   `Tools/MM6/Configure Quest 2 Project Settings`
4. Run:
   `Tools/MM6/Prepare New Sorpigal Quest Test`
5. Run:
   `Tools/MM6/Build Quest 2 APK`
6. Save or accept the default output:
   `unity/MM6OutdoorImporter/Builds/Quest2/MM6Viewer-Quest2.apk`

The Quest test setup creates a build list with only:

- `Assets/MM6Viewer/Scenes/MM6Viewer.unity`
- `Assets/MM6Imported/oute3/Scenes/oute3.unity`

At runtime, `MM6Viewer.unity` launches first and directly opens `oute3` without the map selection screen.

## Install Local Quest APK

If you have built the New Sorpigal Quest 2 APK locally, it will be at:

```text
/Users/kalinbas/Downloads/mm6hd/unity/MM6OutdoorImporter/Builds/Quest2/MM6Viewer-Quest2.apk
```

From the repository root, the same APK is:

```text
unity/MM6OutdoorImporter/Builds/Quest2/MM6Viewer-Quest2.apk
```

To install it on Quest 2, enable Developer Mode on the headset, connect it over USB, accept the headset authorization prompt, then run:

```bash
adb install -r unity/MM6OutdoorImporter/Builds/Quest2/MM6Viewer-Quest2.apk
```

If `adb` reports `device unauthorized`, keep the headset awake and accept the USB debugging confirmation inside the headset before retrying.

The public source repository intentionally does not include the built APK or extracted MM6 game assets. Rebuild the APK locally from a valid MM6 installation before sharing or installing your own build.

Batchmode build on macOS:

```bash
/Applications/Unity/Hub/Editor/6000.4.0f1/Unity.app/Contents/MacOS/Unity \
  -batchmode \
  -quit \
  -projectPath "$(pwd)/unity/MM6OutdoorImporter" \
  -executeMethod Mm6QuestBuild.BuildQuest2Apk \
  -mm6BuildOutput "$(pwd)/unity/MM6OutdoorImporter/Builds/Quest2/MM6Viewer-Quest2.apk" \
  -logFile "$(pwd)/unity/MM6OutdoorImporter/Builds/Quest2/build.log"
```

Close the Unity Editor before using the batchmode command on the same project. If the build fails, inspect:

```text
unity/MM6OutdoorImporter/Builds/Quest2/build.log
```

Install the APK with `adb` after the Quest 2 is in Developer Mode and authorized:

```bash
adb install -r unity/MM6OutdoorImporter/Builds/Quest2/MM6Viewer-Quest2.apk
```

## How To Use Outdoor Imports

1. In Unity Hub, install or select Unity 6 `6000.4.0f1`.
2. Open this folder itself as a Unity project:
   `unity/MM6OutdoorImporter`
3. Import an outdoor package with either:
   `Tools/MM6/Import Outdoor Map Package...`
   or
   `Tools/MM6/Import Bundled Packages In Project`
   For a quick New Sorpigal-only test package that does not rebuild all 15 outdoor maps, run:
   `python3 tools/build_mm6_new_sorpigal_package.py`
   Then use:
   `Tools/MM6/Import Outdoor Map Package...`
   and select:
   `unity/MM6OutdoorImporter/MM6SinglePackages/oute3`
   For Quest 2 testing with only New Sorpigal in the viewer catalog and Build Settings, use:
   `Tools/MM6/Prepare New Sorpigal Quest Test`
4. Open the generated scene:
   `Assets/MM6Imported/<map>/Scenes/<map>.unity`
5. Press Play, then click in the Game view to lock the cursor.

The outdoor importer now also creates an `Environment` rig with `Mm6OutdoorEnvironmentController`. Its defaults are exported into the package, and you can tweak the starting hour, time speed, or forced weather from the Inspector after import. The default time speed now matches MM6/OpenEnroth: game time runs at `30x` real time, so one full in-game day takes `48` real minutes.

You can still import an external export folder with `Tools/MM6/Import Outdoor Map Package...`.

To build the full stitched outdoor world for local testing:

1. Generate all 15 outdoor packages from the original install:
   `python3 tools/build_mm6_outdoor_world_packages.py`
2. In Unity, run:
   `Tools/MM6/Import Stitched Full Outdoor World`
3. Open the generated scene:
   `Assets/MM6Imported/mm6_world/Scenes/mm6_world.unity`
4. Press Play.

The stitched world centers the `5 x 3` MM6 outdoor grid around the origin and keeps only one active outdoor environment and one player spawn, based on New Sorpigal, so it stays usable for local desktop testing.

## How To Use Indoor Imports

1. Export a real decompressed dungeon `.blv`:
   `python3 tools/export_mm6_indoor_to_unity.py --map /path/to/dungeon.blv --output exported/unity-indoor/dungeon`
   For a quick Goblinwatch-only test package from the original MM6 install, run:
   `python3 tools/build_mm6_goblinwatch_package.py`
   Then use:
   `Tools/MM6/Import Indoor Map Package...`
   and select:
   `unity/MM6OutdoorImporter/MM6SingleIndoorPackages/d01`
   For a small varied indoor test set, run:
   `python3 tools/build_mm6_indoor_test_packages.py`
   This builds:
   `d01` Goblinwatch
   `d02` Abandoned Temple
   `d03` Shadow Guild Hideout
   `d05` Snergle's Caverns
   `d11` Corlagon's Estate
   `sewer` Free Haven Sewers
2. Optionally bundle it into the Unity project:
   `python3 tools/bundle_mm6_unity_package.py --input-json exported/unity-indoor/dungeon/map.json --output-dir unity/MM6OutdoorImporter/MM6IndoorPackages/dungeon`
3. In Unity, run either:
   `Tools/MM6/Import Indoor Map Package...`
   or
   `Tools/MM6/Import Bundled Indoor Packages In Project`
4. Open the generated scene:
   `Assets/MM6Imported/dungeon/Scenes/dungeon.unity`
5. Press Play, then click in the Game view to lock the cursor.

## Current Terrain Limitation

If `dtile.bin` is not provided to the exporter, terrain textures fall back to one representative base texture per tileset group. Structures and decorations still use the real extracted textures.

## Current Monster Limitation

Outdoor monster previews are currently driven by exported spawn data plus fallback encounter definitions when exact table data is not available. Grouped outdoor monsters may be spread slightly in Unity so stacked encounters stay visible. Monsters are currently idle-only previews with standing animation where available; pursuit and combat movement are disabled for now.

## Current Animation Coverage

Animated sprite loops are imported from `SFT.txt`, so effects like fires, fountains, flags, sea markers, and the mage standing/flying idle now animate in Unity. I did not find a local MM6 texture-frame table (`TFT`), so model and terrain bitmap animation is still treated as static in this project.

## Current Indoor Limitation

The indoor pipeline is a first pass aimed at real `.blv` dungeon geometry. It currently imports static BLV faces, decorations, lights, spawn points, and idle monster previews from BLV spawn records, but it does not yet apply the corresponding `.dlv` runtime state. That means moving doors, saved door states, exact indoor monster families for maps that rely on DLV-only data, chests, and other location-delta systems are not reconstructed yet. Indoor animated bitmap faces also still depend on `dtft.bin`, so those remain static for now.

## Current Viewer Limitation

Quest support is wired around Unity OpenXR plus a custom VR locomotion rig. The generated Quest test scene skips the map selector and direct-loads the first catalog map; for the New Sorpigal test package that is `oute3`.
